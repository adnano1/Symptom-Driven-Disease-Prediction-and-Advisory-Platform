About Our Team: Your AI & ML Engineers
Meet the Developers

We are a group of three CDAC students building the Symptom-Driven Disease Prediction and Advisory Platform web application - a lightweight ML + Flask system that maps user-reported symptoms to likely disease classes and provides contextual guidance. Our team combines strengths in data preparation, machine learning model training, backend integration, and frontend UI/UX to produce a deployable, explainable prototype.

Our Expertise

Our technical stack includes Python, scikit-learn (SVC), NumPy, Pandas, Flask for the backend, Jinja2 templating, and Bootstrap for the frontend. We focus on interpretability (clear binary symptom encoding + CSV lookup tables), efficient model serialization (pickle), and a clean, responsive user interface suitable for demonstrations and academic evaluation.

A Commitment to Excellence

We follow a reproducible workflow: data curation → preprocessing & feature engineering → model development (SVC) → evaluation (accuracy, precision, recall, F1, confusion matrices) → serialization and Flask deployment. Quality assurance includes cross-validation, unit/integration checks for model↔server integration, and clear documentation of limitations and ethical considerations.

Passion for Health Tech

Our motivation is to create an explainable first-line triage tool for lay users in resource-constrained settings. We emphasise non-prescriptive guidance, transparent model behaviour, and easy deployment so the application can be used for demonstrations, teaching, or pilot testing (not as a clinical diagnostic tool).
